## 評価軸（優先度の判断基準・2026-06-21 ユーザー確定）

中間発表が近い文脈を前提に、以下を最大化する。「デモで効く × 低コスト × 低リスク」を上に置く。

1. **発表デモのインパクト** — 中間発表で見せ場になるか。
2. **運用の安心感** — 本番障害を見逃さない・コストを把握する。
3. **完成度・取りこぼし回収** — 着手済み/残タスクを仕上げて消化する。
4. **採点の説得力・品質** — プロダクトの核。ただし本丸の disfluency は加藤所有。

スコープは**プロダクト全体**（担当を問わず棚卸し）。品質も優先度表に含める。

## 推奨ロードマップ（発表逆算）

`1-b → 4-a → 4-b/4-c → 3-a`、隙間で低コスト回収（1-a close / 3-d）。

1. **物語の軸（推移グラフ 1-b）と華（アニメ 4-a）**でデモを固める。
2. **事故らない堅牢さ（4-c）と待ち時間演出（4-b）**で完成度を上げる。
3. 仕上げに**運用の安心（Sentry 3-a）**。
4. 隙間で**低コスト回収（1-a の #7 close、3-d の auth 反映）**。
